### Java

#### WHY: Java를 사용하는 이유

- PROBLEM: 당시 보급된 C언어는 절차지향이고 소스 코드를 기계어로 컴파일하여 CPU(플랫폼)에 따라 기계어가 대응되도록 프로그래머가 직접 포팅해야 했음
- Java는 플랫폼 독립적인 특징을 가지면서 위의 문제를 해결하는 기술로 사용됨

#### Java의 특징

1. 플랫폼 독립성: JVM으로 인해 CPU가 달라도 바이트 코드를 거쳐 어디서든 실행 가능
2. 보안 안정성이 높음: 기존 어플리케이션이 OS를 거쳐 하드웨어에 영향을 미쳤으나, 자바는 JVM이 샌드박스로 한 번 컴파일하므로 안정성이 높음
3. 메모리 관리에 용이함: Stack/Heap, GC로 메모리를 자동으로 관리함
4. 객체 지향 언어

Java의 단점: VM이 OS로부터 자원을 할당받아 관리하므로 C언어에 비해서 무거움

#### JVM, JRE, JDK

**JVM(Java Virtual Machine)**

- DEF: 자바의 바이트 코드를 기계어로 바꿔 실행하는 엔진
- OS로부터 메모리를 할당받아 관리함

**JRE(Java Runtime Environment)**

- DEF: 자바 프로그램을 실행하기 위한 최소한의 환경
- 구성: JVM + 클래스 라이브러리

**JDK(Java Development Kit)**

- DEF: 자바 프로그램을 실행하고 만들 때 필요한 도구의 모음
- 구성: JRE + 개발용 도구(javac, 디버거)

### Java의 메모리 관리

**C언어**
- 하드웨어에 직접 요청해 메모리를 할당하고 해제할 수 있음
- Pros & Cons
    - Pros: 빠르고 메모리 관리 효율적
    - Cons: 메모리 누수 가능성

#### Stack

- DEF: JVM이 메서드를 실행할 때 사용하는 임시 공간, 기본형 타입(Primitive Type) 변수나 메서드가 저장됨(Heap을 참조하기 위한 주소를 담는 변수 포함)
- 스레드별로 고유한 스택이 생성되며 메서드 호출 시 Stack Frame을 생성하고, 메서드가 종료되면 제거됨
- 자료구조 Stack과 동일하게 LIFO으로 동작

#### Heap

- DEF: 여러 메서드에서 참조가능한 값을 저장하는 공간, 메서드가 종료되어도 객체는 사라지지 않지만 Stack에서 더 이상 참조되지 않는 경우 GC가 발생해 제거됨, 참조 타입(Reference type)의 실제 값이 저장됨
- 구조
    - Young Generation: 새로운 객체가 할당되는 곳, 수명이 짧고 금방 소멸되는 객체가 저장되며, GC가 자주 발생(Minor GC)
        - Eden: 객체가 처음 생성되면 저장되는 공간
        - Survivor 0(S0), 1(S1): Eden에서 GC 후 살아남은 객체를 복사하는 공간
    - Old Generation: Young Generation에서 일정 시간이 지나 살아남은 객체를 저장하는 곳, 수명이 길고 메모리 공간도 넓게 할당되며 GC 빈도가 낮으나 한 번 실행하면 (Major GC) 시간이 오래 걸릴 수 있음
        - Major GC가 발생하면 GC를 위해 어플리케이션 스레드가 일시적으로 멈추는 Stop-the-world 현상이 일어날 수 있음
- WHY
    - Young/Old Generation을 나누는 이유: 생성과 삭제가 잦은 객체와 오랜기간, 크게 할당되는 객체를 할당하고 제거할 때 각각 효율적인 알고리즘을 사용하여 GC의 성능을 최적화하기 위해

#### GC

Garbage Collection

- DEF: Stack에서 더 이상 참조되지 않는 Heap 메모리 영역의 객체를 자동으로 정리하기 위한 메모리 관리 기능

### Java의 Primitive Type vs Reference Type 변수

#### Primitive Type(기본형)

- 실제 값이 저장됨
- null 불가
- 소문자
- 선언과 동시에 Stack 영역에 메모리 공간 할당
- ex. int, boolean

#### Reference Type(참조형)

- 객체가 저장된 주소를 저장
- Stack에 주소를 저장한 변수를 저(Heap 영역에 실제 객체 값을 저장)
- null로 초기화할 수 있음

### String

- DEF: 연속된 문자를 저장하는 불변 객체
- 특징
    - Heap 메모리 영역의 String constant pool에 문자열 저장
    - 같은 문자열의 중복 선언을 방지함
    - new String()으로 선언하는 문자열 리터럴은 기존 문자열과는 다른 객체(==으로 비교시 false반환)
